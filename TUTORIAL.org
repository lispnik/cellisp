Cellisp is a spreadsheet *engine* whose formula language is Common Lisp itself.
There is no formula mini-language and no bundled UI: a formula is an ordinary
Lisp form that happens to read other cells with ~cell~ and ~cells~, and the
engine tracks dependencies, recomputes exactly what changed, catches per-cell
errors, and can persist and render the result.

* Setup: loading the engine

The first block loads the system. It mirrors the pattern used by the other
files under =examples/=: pull in ASDF, register this directory on its source
registry (so the ~ocicl~-installed dependency ~bordeaux-threads~, unpacked under
=./ocicl=, is found), then load the ~cellisp~ system and its optional display
layer. We work inside the ~cellisp~ package and pull in the display symbols so
everything reads unqualified.

#+begin_src lisp
(require :asdf)
(asdf:initialize-source-registry            ; find cellisp + its ocicl deps under ./
 (list :source-registry (list :tree (uiop:getcwd)) :inherit-configuration))
(asdf:load-system "cellisp")
(asdf:load-system "cellisp/display")

(in-package #:cellisp)
(use-package '#:cellisp/display)
#+end_src

Two tiny helpers, used throughout. ~show~ just prints a labelled value; ~check~
prints *and verifies* it, tallying any mismatch so the whole run fails loudly if
a claimed output ever changes.

#+begin_src lisp
(defparameter *checks* 0)
(defparameter *failures* 0)

(defun show (label value)
  "Print LABEL => VALUE (display only; note it captures just the primary value)."
  (format t "~&~28A => ~S~%" label value))

(defun check (label got expected)
  "Print LABEL => GOT and verify GOT is EQUAL to EXPECTED, tallying failures."
  (incf *checks*)
  (format t "~&~28A => ~S~%" label got)
  (unless (equal got expected)
    (incf *failures*)
    (format *error-output* "~&    !! FAIL ~A: expected ~S~%" label expected)))

(format t "~&==== Cellisp tutorial ====~%")
#+end_src

* Cells and formulas

A *sheet* holds cells addressed in A1 style. ~make-sheet~ creates an empty one;
~set-cell~ installs one formula; ~set-cells~ installs several at once and
recomputes them together (so they may reference each other in any order). A
formula is just a Lisp form — a bare number is a formula that evaluates to
itself, and ~(sum (cells "A1" "A3"))~ reads a range and adds it up.

Note the binding syntax: ~set-cells~ takes ~(designator formula)~ *two-element
lists*, e.g. ~("A1" 10)~.

#+begin_src lisp
(defparameter *s* (make-sheet))

(set-cells *s*
  '(("A1" 10)
    ("A2" 20)
    ("A3" 30)
    ("A4" (sum (cells "A1" "A3")))))

(check "A4 = sum(A1:A3)" (get-value *s* "A4") 60)
#+end_src

~get-value~ returns two values — the value and an error (or ~NIL~). Here it is
~60~; the formula ran as Lisp, ~cells~ returned ~(10 20 30)~ and ~sum~ added
them. (Called for one value in argument position, as ~check~ does, only the
primary value comes through — the error is the *second* value.)

Because formulas are real Lisp, they are not limited to arithmetic — a formula
can call any function, including one you define, and can return any value, not
just a number:

#+begin_src lisp
(defun celsius->fahrenheit (c) (+ 32 (* c 9/5)))

(set-cells *s*
  '(("C1" 100)
    ("C2" (celsius->fahrenheit (cell "C1")))       ; call an ordinary defun
    ("C3" (mapcar #'evenp (cells "A1" "A3")))))     ; a formula returning a list
(check "100C in F"        (get-value *s* "C2") 212)
(check "evens in A1:A3"   (get-value *s* "C3") '(t t t))
#+end_src

That is the whole pitch: no formula DSL to outgrow, because the formula language
is the programming language.

* Recalculation and errors

Edit an input and every dependent recomputes automatically — you never call a
"recalculate" button, and only the affected closure is swept.

#+begin_src lisp
(set-cell *s* "A1" 100)
(check "A4 after A1 := 100" (get-value *s* "A4") 150)
#+end_src

Errors are per-cell values, not crashes. A cell whose formula signals stores
the condition; ~get-value~ hands it back as the second value while the rest of
the sheet keeps working.

#+begin_src lisp
(set-cells *s* '(("B1" (/ 1 0))))
(multiple-value-bind (val err) (get-value *s* "B1")
  (format t "~&B1: value=~S  error=~A~%" val (type-of err)))
#+end_src

That prints ~value=NIL error=CELL-EVAL-ERROR~ — the division-by-zero was
caught and wrapped. Fix the cell and it recovers:

#+begin_src lisp
(set-cell *s* "B1" '(/ 10 2))
(check "B1 repaired" (get-value *s* "B1") 5)
#+end_src

*Gotcha — empty cells.* Reading a *single* empty cell with ~(cell "Z9")~
signals ~unbound-cell~, but reading a *range* treats gaps as ~NIL~ so partly
filled ranges still add up. When a range may also contain *errored* cells, use
~safe-cells~, which omits both blanks and errors so an aggregate still works:

#+begin_src lisp
(set-cells *s* '(("E1" 5) ("E2" (/ 1 0)) ("E3" 7)))   ; E2 is broken
(set-cell  *s* "E4" '(sum (safe-cells "E1" "E3")))     ; tolerates the hole
(check "sum(safe E1:E3)" (get-value *s* "E4") 12)      ; 5 + 7, E2 skipped
#+end_src

Also: ~set-cell~ re-signals its own cell's error (so a mistake surfaces
immediately at the REPL); in a non-interactive script, wrap it in ~ignore-errors~
if a deliberately-broken cell would otherwise abort the run. ~set-cells~ and
~get-value~ never re-signal.

* Seeing the dependency graph: =explain=

Dependency tracking is the engine's whole job, and you can *see* it. ~explain~
walks a cell's precedents and prints an indented tree — each node showing the
cell, its current value, and its formula. Let's give ~A4~ a dependent so the
tree has depth:

#+begin_src lisp
(set-cell *s* "A5" '(* (cell "A4") 2))
(check "A5 = A4 * 2" (get-value *s* "A5") 300)

(format t "~&--- explain A5 ---~%")
(explain *s* "A5")
#+end_src

#+begin_example
--- explain A5 ---
A5 = 300   (* (CELL "A4") 2)
└─ A4 = 150   (SUM (CELLS "A1" "A3"))
   ├─ A1 = 100   100
   ├─ A2 = 20   20
   └─ A3 = 30   30
#+end_example

For an *errored* cell the tree follows the failing branch down to the deepest
cell that actually broke — the root cause — which makes it a debugger for
formulas. (For programmatic access to the same edges, ~precedents~ and
~dependents~ return a cell's neighbours as ~(row . col)~ conses.)

* Names and the environment

Cells and ranges can be *named*, and a sheet can carry an *environment* of
constants that formulas see as ordinary lexical variables. Names must exist
before a formula uses them.

#+begin_src lisp
(defparameter *env-sheet*
  (make-sheet :environment '((tax-rate . 8/100))))     ; an exact rational

(set-cell *env-sheet* "A1" 250)                  ; a subtotal
(set-name *env-sheet* "subtotal" "A1")           ; name the cell
(set-cell *env-sheet* "A2" '(* (cell "subtotal") tax-rate))

(check "tax on subtotal" (get-value *env-sheet* "A2") 20)   ; 250 * 8/100
#+end_src

Inside the formula, ~tax-rate~ is bound from the environment and the named cell
is read with ~(cell "subtotal")~ — ~cell~ resolves names, A1 refs, and
~"Sheet!A1"~ cross-sheet refs alike. (~name-ref~ exists too, but it returns the
cell's *reference* — its coordinates — for programmatic use, not its value.)
Note the environment constant is an *exact rational* ~8/100~: formulas do exact
arithmetic unless you hand them floats, so ~250 * 8/100~ is exactly ~20~, no
rounding.

Ranges get names too, which the stdlib aggregates accept directly:

#+begin_src lisp
(set-cells *env-sheet* '(("D1" 3) ("D2" 4) ("D3" 5)))
(set-range *env-sheet* "scores" "D1" "D3")
(set-cell *env-sheet* "D4" '(sum (cells "scores")))
(check "sum(scores)" (get-value *env-sheet* "D4") 12)
#+end_src

* The formula standard library

Formulas are Lisp, so the whole language is available — but a small stdlib adds
the spreadsheet idioms. The range-reading ones (~sum~, ~average~, ~cnt~,
~grid~, ~cells~, ~safe-cells~) must run *inside* a formula, where the current
sheet is bound; the pure helpers work anywhere.

#+begin_src lisp
(check "minimum" (minimum 3 1 2) 1)
(check "maximum" (maximum 3 1 2) 3)
(check "sortv"   (sortv '(3 1 2)) '(1 2 3))
(check "iferror" (iferror (elt #() 5) :fallback) :fallback)
#+end_src

~iferror~ is a macro: it evaluates its form and, if that signals, yields the
fallback instead — the spreadsheet ~IFERROR(...)~ idiom.

The range aggregates ~average~ and ~cnt~, and the predicate-filtered family
~countif~/~sumif~/~averageif~, all read live cells from inside a formula:

#+begin_src lisp
(defparameter *agg* (make-sheet))
(set-cells *agg*
  '(("A1" 3) ("A2" 4) ("A3" 5)
    ("B1" (average (cells "A1" "A3")))
    ("B2" (cnt (cells "A1" "A3")))
    ("B3" (countif #'evenp (cells "A1" "A3")))
    ("B4" (sumif #'oddp (cells "A1" "A3")))))
(check "average(A1:A3)" (get-value *agg* "B1") 4)
(check "cnt(A1:A3)"     (get-value *agg* "B2") 3)
(check "countif evenp"  (get-value *agg* "B3") 1)   ; just 4
(check "sumif oddp"     (get-value *agg* "B4") 8)   ; 3 + 5
#+end_src

~to-number~ coerces text to a number. Its convention is *explicit*: you tell it
which character is the decimal point and (optionally) the grouping separator, so
it never has to guess between "1,5" meaning one-and-a-half (many locales) versus
fifteen-hundred. Note the wart: the ~default~ is a positional ~&optional~, so it
must come *before* the keywords.

#+begin_src lisp
(check "to-number strict"   (to-number "42") 42)
(check "to-number euro"     (to-number "1,5" nil :decimal #\,) 1.5)
(check "to-number grouped"  (to-number "1,234.56" nil :group #\,) 1234.56)
(check "to-number bad"      (to-number "oops" :na) :na)
#+end_src

~vlookup~ searches a table (a list of rows) and returns a column from the
matching row; ~grid~ reads a rectangular block of live cells as such a table:

#+begin_src lisp
(check "vlookup"
       (vlookup "banana" '(("apple" 3) ("banana" 5) ("cherry" 8)) 2)
       5)

(defparameter *prices* (make-sheet))
(set-cells *prices*
  '(("A1" "apple")  ("B1" 3)
    ("A2" "banana") ("B2" 5)
    ("A3" "cherry") ("B3" 8)
    ("D1" (vlookup "cherry" (grid "A1" "B3") 2))))
(check "vlookup over grid" (get-value *prices* "D1") 8)
#+end_src

* Whole-column and whole-row references

Sometimes you want *all* of a column, not a bounded range — a total that keeps
working as rows are added. ~(col "A")~ reads every populated cell of column A;
~(row 5)~ reads a whole row; bands span several (~(col "A" "C")~), and the Excel
colon form works through ~cells~ (~(cells "A:A")~, ~(cells "1:1")~).

#+begin_src lisp
(defparameter *cs* (make-sheet))
(set-cells *cs* '(("A1" 10) ("A2" 20) ("A3" 30)))
(set-cell *cs* "C1" '(sum (col "A")))
(check "sum(col A)"          (get-value *cs* "C1") 60)

(set-cell *cs* "A9" 40)                     ; a cell added LATER, further down column A
(check "col A picks up A9"   (get-value *cs* "C1") 100)

(set-cell *cs* "C2" '(sum (cells "A:A")))   ; the Excel colon form, same result
(check "sum(A:A)"           (get-value *cs* "C2") 100)
#+end_src

The key point is the second check: ~C1~ re-totals when ~A9~ is filled in *later*,
even though it was empty when the formula was written. A whole-column reference
tracks the column as a whole — it records *one* coarse dependency, not an edge per
cell, so there is no per-cell edge explosion and future cells count automatically.
A whole *row* reads the same way:

#+begin_src lisp
(defparameter *rs* (make-sheet))
(set-cells *rs* '(("A1" 5) ("B1" 6) ("C1" 7)))
(set-cell *rs* "A3" '(sum (row 1)))          ; the reader sits OFF the row it reads
(check "sum(row 1)"         (get-value *rs* "A3") 18)
#+end_src

Whole-column/row references shift correctly under structural edits, too: deleting an
endpoint of a band shrinks it (Excel-style), and deleting the whole column leaves a
~#REF!~.

* Tables

A *table* is a named header'd rectangle whose columns you reference by *header text*
rather than grid coordinate — so the reference follows the column's data as the
table grows, shrinks, or moves. ~set-table~ names the region (its first row is the
header row); ~table-col~ (or the Excel string ~Name[Column]~ through ~cells~) reads
a column's data rows.

#+begin_src lisp
(defparameter *ts* (make-sheet))
(set-cells *ts*
  '(("A1" "Region") ("B1" "Qty") ("C1" "Amount")
    ("A2" "North")  ("B2" 3)     ("C2" 100)
    ("A3" "South")  ("B3" 10)    ("C3" 250)))
(set-table *ts* "Sales" "A1" "C3")           ; row 1 = the header row
(set-cell *ts* "E1" '(sum (table-col "Sales" "Amount")))
(set-cell *ts* "E2" '(sum (cells "Sales[Amount]")))   ; Excel string form
(check "sum(Sales[Amount])" (get-value *ts* "E1") 350)
(check "sum via string form" (get-value *ts* "E2") 350)
#+end_src

The read covers the column's *data* rows only — the header row is excluded, so the
sum is 350 (not the text "Amount" plus the numbers). Because ~table-col~ records the
same coarse whole-column dependency as ~col~, the total re-fires as the data changes.
And typing a cell (a ~set-cell~) directly below the table *auto-expands* it:

#+begin_src lisp
(set-cell *ts* "A4" "East") (set-cell *ts* "B4" 5) (set-cell *ts* "C4" 175)
(check "table grew to A1:C4" (table-ref *ts* "Sales") '((0 . 0) . (3 . 2)))
(check "total auto-includes the new row" (get-value *ts* "E1") 525)   ; 100+250+175
#+end_src

Two more conveniences. A *calculated column* reads its own row with ~:this-row~ (the
~Name[@Column]~ form) — the cell in a column on the *computing* cell's row:

#+begin_src lisp
(defparameter *cts* (make-sheet))
(set-cells *cts*
  '(("A1" "Item") ("B1" "Qty") ("C1" "Price") ("D1" "Total")
    ("A2" "Pen")  ("B2" 3)     ("C2" 2)
    ("A3" "Pad")  ("B3" 5)     ("C3" 4)))
(set-table *cts* "Inv" "A1" "D3")
(set-cells *cts*
  '(("D2" (* (table-col "Inv" "Qty" :this-row) (table-col "Inv" "Price" :this-row)))
    ("D3" (* (cells "Inv[@Qty]") (cells "Inv[@Price]")))))   ; string @ form
(check "Total = Qty*Price, row 2" (get-value *cts* "D2") 6)
(check "Total = Qty*Price, row 3" (get-value *cts* "D3") 20)
#+end_src

And a *totals row* (~:totals t~) is excluded from data reads — so a
~(sum (table-col …))~ placed *in* the totals row sums the data without counting
itself, no cycle:

#+begin_src lisp
(defparameter *tot* (make-sheet))
(set-cells *tot*
  '(("A1" "R") ("B1" "Amount") ("A2" "N") ("B2" 100) ("A3" "S") ("B3" 250) ("A4" "Total")))
(set-table *tot* "T" "A1" "B4" :totals t)    ; row 4 is the totals row
(set-cell *tot* "B4" '(sum (table-col "T" "Amount")))
(check "totals cell sums the data" (get-value *tot* "B4") 350)
#+end_src

Tables serialize, shift Excel-faithfully under ~insert~/~delete~, and reject overlap;
the registration API is ~set-table~, ~table-ref~, ~remove-table~, ~map-tables~,
~table-at~.

* Cell behaviours: observers, caching, validation

Cells are CLOS objects and behaviours are mixins layered on in place. We reuse
~*s*~ from the earlier sections (its ~A4 = sum(A1:A3)~ is still live). ~observe~
fires a callback whenever a cell's value changes:

#+begin_src lisp
(defparameter *log* '())
(observe *s* "A4" (lambda (new-value) (push new-value *log*)))

(set-cell *s* "A1" 7)     ; A4 = 7 + 20 + 30 = 57, so the observer fires
(check "observer log" *log* '(57))
#+end_src

The callback is a function of the new value; it ran because editing A1 forced
A4 to recompute. ~set-cached~ marks a cell so its formula is reused until an
input actually changes — it does not change the value, only how often the
formula re-runs:

#+begin_src lisp
(set-cached *s* "A4" t)
(check "A4 (cached)" (get-value *s* "A4") 57)
#+end_src

A *validator* rejects values a predicate fails. The rejection signals
~invalid-value~ (which ~set-cell~ re-signals, so wrap it), and the cell is left
holding that error — which, as the next section shows, renders as ~#VALUE!~:

#+begin_src lisp
(defparameter *val* (make-sheet))
(set-cell *val* "A1" 10)
(set-validator *val* "A1" #'plusp)             ; only positives allowed
(ignore-errors (set-cell *val* "A1" -5))       ; rejected
(multiple-value-bind (v e) (get-value *val* "A1")
  (check "rejected value" v nil)
  (check "rejection token" (error-token e) "#VALUE!"))
#+end_src

Related toggles in the same family: ~set-frozen~ (make a cell read-only),
~set-volatile~ (always recompute), and ~set-ttl~ (expire a cached value after a
time).

** Writing your own behaviour mixin

The built-in behaviours are not a closed set. A mixin is just a class that
overrides one of the cell generics — ~compute-value :around~ to shape the value,
~cell-swept :after~ to react to a settled recompute, ~cell-writable-p~ to guard
writes — and ~combined-class~ already knows how to layer any class you name onto
a cell. ~register-mixin~ makes yours behave exactly like a built-in: it joins the
known-mixin set (so it survives later morphs and ~mixins-at~ reports it) and,
given ~:dump~/~:restore~, round-trips through persistence.

Here is a value-shaping mixin that scales a cell's computed value by a per-cell
factor:

#+begin_src lisp
(defclass scale-mixin ()
  ((factor :initform 1 :accessor scale-factor)))

(defmethod compute-value :around ((cell scale-mixin) sheet ref)
  (let ((v (call-next-method)))               ; the value the cell would have had
    (if (numberp v) (* (scale-factor cell) v) v)))

(register-mixin 'scale-mixin
                :serialize-as :scale
                :dump    (lambda (c) (scale-factor c))
                :restore (lambda (c state) (setf (scale-factor c) state)))
#+end_src

~set-mixin~ attaches it by designator — the general form of ~set-cached~ &
friends — and ~mixins-at~ lists what a cell carries:

#+begin_src lisp
(defparameter *ext* (make-sheet))
(set-cell *ext* "A1" 21)
(set-cell *ext* "B1" '(cell "A1"))
(set-mixin *ext* "B1" 'scale-mixin)
(setf (scale-factor (find-cell *ext* (parse-ref "B1"))) 2)
(recalc *ext* "B1")
(check "scaled value"  (get-value *ext* "B1") 42)
(check "mixins on B1"  (mixins-at *ext* "B1") '(scale-mixin))
#+end_src

Because we registered a serializer, the factor survives a save/load round-trip —
the mixin is re-added and its state restored before the reloaded formulas
recompute, so the value comes back shaped:

#+begin_src lisp
(let ((r (form->sheet (sheet->form *ext*))))
  (check "scale survives reload" (get-value r "B1") 42))
#+end_src

Formulas themselves are just as open: they are ordinary Lisp forms read in the
~cellisp~ package, so any function you can name is fair game — define your own and
call it (bare if it lives in ~cellisp~, or ~your-package:fn~ otherwise) right
inside a cell.

* Editing: undo, transactions, structure, spilling

Edits are undoable, can be grouped atomically, and include structural
operations. Start a fresh sheet:

#+begin_src lisp
(defparameter *edit* (make-sheet))
(set-cell *edit* "A1" 1)
(set-cell *edit* "A1" 2)
(undo *edit*)
(check "A1 after undo" (get-value *edit* "A1") 1)
(redo *edit*)
(check "A1 after redo" (get-value *edit* "A1") 2)
#+end_src

~with-transaction~ batches several edits into one sweep and one undo step:

#+begin_src lisp
(with-transaction (*edit*)
  (set-cell *edit* "B1" 10)
  (set-cell *edit* "B2" 20)
  (set-cell *edit* "B3" '(sum (cells "B1" "B2"))))
(check "B3 (transaction)" (get-value *edit* "B3") 30)
#+end_src

~insert-row~ shifts cells (and the references pointing at them) down, and
~copy-cell~ copies a formula with its references translated:

#+begin_src lisp
(insert-row *edit* 1)                     ; everything moves down one row
(check "A1 shifted to A2" (get-value *edit* "A2") 2)
(copy-cell *edit* "A2" "C5")
(check "C5 (copied)" (get-value *edit* "C5") 2)
#+end_src

A *spill* writes a computed sequence across a block from an anchor. A flat list
spills down a column; ~respill~ re-fills and clears any shrunk leftover:

#+begin_src lisp
(check "spill extent" (spill *edit* "E1" '(list 100 200 300)) '(3 . 1))
(check "E2 from spill" (get-value *edit* "E2") 200)
#+end_src

* Workbooks and cross-sheet references

A *workbook* groups named sheets; a formula reaches another sheet with a
~"Sheet!A1"~ reference.

#+begin_src lisp
(defparameter *wb* (make-workbook))
(defparameter *sales*   (add-sheet *wb* "Sales"))
(defparameter *summary* (add-sheet *wb* "Summary"))

(set-cells *sales* '(("A1" 100) ("A2" 200)))
(set-cell  *summary* "A1" '(+ (cell "Sales!A1") (cell "Sales!A2")))

(check "cross-sheet total" (get-value *summary* "A1") 300)
#+end_src

Editing =Sales= flows straight through to =Summary= — cross-sheet dependencies
are tracked just like local ones:

#+begin_src lisp
(set-cell *sales* "A1" 150)
(check "total after edit" (get-value *summary* "A1") 350)
#+end_src

* The display layer

The core engine stores values, not appearances. The optional ~cellisp/display~
system turns values and errors into strings: ~display-value~ for one cell,
~print-sheet~ for a whole grid, plus a *format registry* built with
~make-formats~ that styles columns and adds conditional rules. Errors render as
familiar spreadsheet tokens via ~error-token~.

#+begin_src lisp
(defparameter *report* (make-sheet))
(set-cells *report*
  '(("A1" "Widget") ("B1" 1200.5) ("C1" 0.23)
    ("A2" "Gadget") ("B2" 890)    ("C2" -0.05)))

(defparameter *fmt* (make-formats))
(set-column-format *fmt* "B" '(:currency "$" 2))       ; B as money
(set-column-format *fmt* "C" '(:percent 1))            ; C as a percentage
(add-conditional *fmt* (lambda (v) (and (realp v) (minusp v)))
                 "LOSS" :column "C")                    ; negatives -> "LOSS"

(check "B1 formatted" (display-value *report* "B1" :formats *fmt*) "$1200.50")
(check "C2 formatted" (display-value *report* "C2" :formats *fmt*) "LOSS")
#+end_src

Errored cells render as tokens. Because ~set-cell~ re-signals its own cell's
error, wrap the deliberately-broken cell in ~ignore-errors~ in a script:

#+begin_src lisp
(ignore-errors (set-cell *report* "D1" '(/ 1 0)))
(multiple-value-bind (v e) (get-value *report* "D1")
  (declare (ignore v))
  (check "D1 as token" (error-token e) "#DIV/0!"))
#+end_src

~print-sheet~ lays the whole thing out as an aligned grid — values by default,
or the formulas behind them:

#+begin_src lisp
(format t "~&--- values ---~%")
(print-sheet *report* :formats *fmt* :name "Report")

(format t "~&--- formulas ---~%")
(print-sheet *report* :formulas t :name "Report")
#+end_src

which prints:

#+begin_example
--- values ---
Report
  | A      | B        | C     | D       |
--+--------+----------+-------+---------+
1 | Widget | $1200.50 | 23.0% | #DIV/0! |
2 | Gadget |  $890.00 |  LOSS |         |
--- formulas ---
Report
  | A      | B      | C     | D        |
--+--------+--------+-------+----------+
1 | Widget | 1200.5 | 0.23  | =(/ 1 0) |
2 | Gadget | 890    | -0.05 |          |
#+end_example

* Persistence

~save-sheet~ / ~load-sheet~ (and ~save-workbook~ / ~load-workbook~) serialize
the declarative model — formulas, names, environment, notes. *Values are
recomputed on load*, not stored, so a reloaded sheet is always internally
consistent. (Closures serialize only as *named* functions; anonymous
lambdas don't round-trip.)

#+begin_src lisp
(defparameter *sheet-path*
  (merge-pathnames "cellisp-tutorial.sheet" (uiop:temporary-directory)))

(defparameter *persist* (make-sheet))
(set-cells *persist*
  '(("A1" 21) ("A2" 2) ("A3" (* (cell "A1") (cell "A2")))))
(save-sheet *persist* *sheet-path*)

(let ((reloaded (load-sheet *sheet-path*)))
  (check "A3 recomputed on load" (get-value reloaded "A3") 42))
#+end_src

Workbooks round-trip with their cross-sheet references intact:

#+begin_src lisp
(defparameter *wb-path*
  (merge-pathnames "cellisp-tutorial-wb.sheet" (uiop:temporary-directory)))
(save-workbook *wb* *wb-path*)

(let ((wb2 (load-workbook *wb-path*)))
  (check "reloaded Summary!A1"
         (get-value (find-sheet wb2 "Summary") "A1") 350))
#+end_src

* Capstone: a small invoice

Tying it together — an environment constant (the tax rate), formulas, a named
range, a note, and a formatted render:

#+begin_src lisp
(defparameter *invoice* (make-sheet :environment '((tax-rate . 0.0875))))
(set-cells *invoice*
  '(("A1" "Item")     ("B1" "Qty") ("C1" "Price") ("D1" "Line")
    ("A2" "Notebook") ("B2" 3)  ("C2" 4.5)  ("D2" (* (cell "B2") (cell "C2")))
    ("A3" "Pen")      ("B3" 10) ("C3" 1.25) ("D3" (* (cell "B3") (cell "C3")))
    ("A4" "Marker")   ("B4" 4)  ("C4" 2.0)  ("D4" (* (cell "B4") (cell "C4")))
    ("A6" "Subtotal") ("D6" (sum (cells "D2" "D4")))
    ("A7" "Tax")      ("D7" (* (cell "D6") tax-rate))
    ("A8" "Total")    ("D8" (+ (cell "D6") (cell "D7")))))
(set-range *invoice* "line-items" "D2" "D4")
(set-note  *invoice* "D7" "state sales tax")     ; a note, retrievable + saved
(check "note on D7" (cell-note *invoice* "D7") "state sales tax")

(defparameter *invoice-fmt* (make-formats))
(set-column-format *invoice-fmt* "C" '(:currency "$" 2))
(set-column-format *invoice-fmt* "D" '(:currency "$" 2))

(format t "~&--- invoice ---~%")
(print-sheet *invoice* :formats *invoice-fmt* :name "Invoice")
(show "invoice total (float)" (get-value *invoice* "D8"))   ; 36.975
#+end_src

#+begin_example
--- invoice ---
Invoice
  | A        | B   | C     | D      |
--+----------+-----+-------+--------+
1 | Item     | Qty | Price | Line   |
2 | Notebook |   3 | $4.50 | $13.50 |
3 | Pen      |  10 | $1.25 | $12.50 |
4 | Marker   |   4 | $2.00 |  $8.00 |
5 |          |     |       |        |
6 | Subtotal |     |       | $34.00 |
7 | Tax      |     |       |  $2.97 |
8 | Total    |     |       | $36.97 |
  names: LINE-ITEMS=D2:D4
  env:   TAX-RATE=0.0875
#+end_example

Two things worth noting in that render. The footer lists the sheet's *names* and
*environment* (the note we attached to D7 doesn't print here, but it is stored,
readable with ~cell-note~, and serialized). And the tax line shows
~$2.97~ while the underlying value is ~34.0 * 0.0875 = 2.975~: the ~0.0875~
here is a *float*, and ~2.975~ isn't exactly representable, so the two-decimal
render rounds down. Feed the formula an exact rational instead and the arithmetic
is exact:

#+begin_src lisp
(check "exact tax (34 * 7/80)" (* 34 7/80) 119/40)   ; 119/40 = 2.975, exactly
#+end_src

That is the whole engine in miniature: cells holding Lisp formulas, automatic
dependency-driven recomputation you can ~explain~, per-cell error values, names
and environments, undo/transactions/spills, workbooks, a display layer, and
persistence — every line of which just ran, and was *checked*, to produce the
output above.

#+begin_src lisp
(format t "~&==== ~D checks, ~D failures ====~%" *checks* *failures*)
(format t "~&==== tutorial complete ====~%")
(when (plusp *failures*)
  (uiop:quit 1))                 ; fail the run if any claimed output changed
#+end_src
