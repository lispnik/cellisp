# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A spreadsheet *backend* whose formula language is Common Lisp itself. There is no UI and no parser for a formula DSL: a cell's formula is either a literal (number/string/etc.) or an arbitrary Lisp form, and cells are read from within formulas via the `cell`/`cells` operators. The system's job is dependency tracking and incremental recalculation.

## Commands

Load and test via ASDF (adjust the implementation as needed; examples use SBCL):

```lisp
;; load the system
(asdf:load-system "cellisp")

;; run the test suite (defsystem test-op perform hook)
(asdf:test-system "cellisp")
```

From the shell:

```bash
sbcl --eval '(asdf:test-system "cellisp")' --quit
```

The project must be visible to ASDF (e.g. symlinked into `~/quicklisp/local-projects/` or `~/common-lisp/`, or registered on `asdf:*central-registry*`).

Tests are a hand-rolled harness in `test.lisp` — no external test framework. `run-tests` prints `N checks, M failures.` and signals an `error` if any check fails. There is no per-test runner; add a `check`/`check-signals` form to `run-tests` to test one thing. There is also a **property-based** check, `property-incremental=full`: over many random acyclic sheets and edit sequences — mixing formula edits, `insert-row`/`delete-row`, `copy-cell`, and `undo` (a deterministic LCG, so failures reproduce) — it asserts the invariant that the sheet's values always equal a full `recalc-all`. This guards the propagation short-circuit and that every editing operation leaves the sheet consistent. Two more properties: `property-serialization-roundtrip` (a random sheet write-then-read reproduces every value/error, plus names and notes) and `property-workbook-incremental=full` (random acyclic multi-sheet workbooks: the incremental cross-sheet cascade always equals a full `recompute-workbook`). The display system has its own `property-display-total` (`display-value` returns a string for every cell of many random error-filled sheets, never signaling). `cells-snapshot` excludes pure dependency-placeholder cells (no formula/value/error) since their exact set legitimately differs between an incrementally-edited sheet and a freshly-recomputed one.

`bench.lisp` is a standalone performance harness (not part of `test-system`): `sbcl --script bench.lisp [scale]` loads the system and prints a timing table, including a direct before/after measurement of the propagation short-circuit (same edits, pruned cone vs. full propagate).

`cellisp/display` is an **optional secondary system** (`display.lisp`, package `#:cellisp/display`, tests in `display-test.lisp` → `(asdf:test-system "cellisp/display")`). It's a pure rendering layer over the public API — `display-value`/`error-token`/`format-value` + an in-memory `make-formats` registry (static per-cell/per-column specs plus value-dependent `add-conditional` rules) — turning cell values and stored conditions into display strings and spreadsheet error tokens. `print-sheet`/`print-workbook` render an aligned text grid to a stream (default stdout); `:formulas t` switches to a "show-formulas" view (`formula-string` prints `=<form>` on one line, in the cellisp package) instead of values. Format keys and conditional rules may be **sheet-qualified** (`"Sales!D5"`, `"Sales!B"`, `:sheet "Summary"`) — stored under `(sheet-name . ref/col)` keys — so one registry styles a whole workbook per sheet; `display-value` passes the sheet's name so `format-for`/`conditional-spec` prefer the most specific scope. The core engine does **not** depend on it; keep display concerns out of core. Because the engine has no distinct `#REF!`/`#NAME?` condition classes, `error-token` disambiguates those by inspecting the condition's report text (see the comment there). CI (`.github/run-tests.lisp`) runs both `cellisp` and `cellisp/display` test systems.

## Architecture

Files load in the `:serial t` order declared in `cellisp.asd`: `package` → `cell` → `sheet` → `workbook` → `eval` → `stdlib` → `api` → `taxonomy` → `serialize` → `edit` → `explain`. Everything is in the single `#:cellisp` package (nickname `#:sheet`). The system depends on `bordeaux-threads` (per-sheet locking).

**References** (`cell.lisp`) — A ref is a `(row . col)` cons of zero-based integers. The external form is an A1 string (`"AA10"`). `parse-ref` coerces a ref cons, string, or symbol into a ref (validating either form and signaling `sheet-error` on anything malformed — bad column letters, a zero/negative row, trailing junk, or a cons that isn't two non-negative integers); `ref-string` is the inverse. Refs are compared with `EQUAL`, which is why the sheet's cell table and every dependency set uses `:test 'equal` / `:test #'equal` throughout — keep that invariant when touching those structures. A cell can also carry a **name**: `set-name`/`name-ref`/`remove-name` register an alias in the sheet's `names` table, and `resolve-ref` (used by `cell`/`cells`) consults it before A1 parsing (skipped when the sheet has no names). The same `names` table also holds **named ranges** (`set-range`/`range-ref`): a single-cell name's value is a ref `(row . col)`, a range name's value is a `(tl-ref . br-ref)` cons — told apart by `%range-value-p` (is the car a cons?). `cells` takes `(top-left &optional bottom-right)`: with one argument it expands a range name to its two corners, or treats a single cell as a 1×1 range; `resolve-ref` of a range name yields its top-left. Both name kinds are serialized (single as `(name . "A1")`, range as `(name "A1" "B3")`) and follow their cells across structural edits (`shift-name-table` shifts both corners of a range). Name resolution is centralized in `resolve-ref-in (sheet designator)` (sheet.lisp) — a name takes precedence over an A1 parse, a range name yields its top-left. `resolve-ref` (in a formula) is just `resolve-ref-in` on the dynamic `*sheet*`; the **public API** (`get-value`/`set-cell`/`clear-cell`/notes/attributes/mixin drivers) calls `resolve-ref-in sheet …` instead of `parse-ref`, so a cell name works anywhere an A1 ref does, not only inside formulas.

**Cell class & sheet struct** (`cell.lisp`, `sheet.lisp`) — `cell` is a **CLOS class** (accessors keep the `cell-` prefix: `cell-value`, `cell-precedents`, …); it caches its `value` (or an `err` condition when evaluation failed) alongside its `formula`, plus two adjacency lists: `precedents` (refs it reads) and `dependents` (refs that read it), and memoizes a `compiled` thunk with the `compiled-from` formula it was built from (see `eval-formula` below). The taxonomy (in `taxonomy.lisp`) is split along orthogonal axes so kinds **compose**:

- **Value source** (mutually exclusive; override `compute-value`): base `cell` (formula), `external-cell` (a Lisp thunk), `async-cell` (non-blocking; value pushed in via `deliver-async`).
- **Behavior mixins** (composable, listed in `*mixin-classes*`), grouped by the seam each hooks:
  - *primary `cell-swept`* — `observable-mixin` (`observe`/`unobserve`) fires subscriber callbacks post-sweep.
  - *`:after cell-swept`* (run alongside the primary and each other via method combination) — `logged-mixin` (`set-logged`/`cell-log`, optional `:limit`) records history; `debounced-mixin` (`debounce`) coalesces a burst into one trailing fire via an injectable scheduler + generation counter; `throttled-mixin` (`throttle`) is its leading-edge twin; `threshold-mixin` (`on-threshold`) fires on crossing a level; `stats-mixin` (`set-stats`/`cell-stats`) keeps running count/sum/min/max/mean; `persisted-mixin` (`set-persist`) calls an output sink on change.
  - *`:around compute-value`* (wrap value production; chain via `call-next-method`) — `cached-mixin` (`set-cached`) memoizes by precedent change, re-noting precedents on a hit so links survive; `ttl-cached-mixin` (`set-ttl`) memoizes by time (injectable clock); `default-mixin` (`set-default`) falls back to a default on error; `transformed-mixin` (`set-transform`) post-processes the value; `validated-mixin` (`set-validator`) signals `invalid-value` on a bad value; `timed-mixin` (`set-timed`/`cell-timing`) profiles run time; `retry-mixin` (`set-retry`) re-runs on transient errors.
  - *`cell-writable-p`* (mutation guards; chain via `call-next-method` so several AND together, `readonly`'s nil short-circuiting) — `readonly-mixin` (`set-readonly`) locks reassignment; `append-only-mixin` (`set-append-only`) is write-once; `typed-input-mixin` (`set-typed-input`) rejects a new formula failing a predicate. Blocked writes signal `readonly-cell`.
  - *`note-set` `:after`* (pre-set hook) — `versioned-mixin` (`set-versioned`/`cell-versions`) records the formula-edit history; `audited-mixin` (`set-audited`/`cell-audit`) records full provenance — `(:time :actor :formula)` per edit, taking the author from `with-actor` (dynamic `*actor*`) and the timestamp from the injectable `*audit-clock*`.

  Three composition styles are in play: *different* generics, the *same* generic via method combination (`:after`, or `:around` chaining), and *wrapping* (`:around`). Any subset stacks — `combined-class` builds `A+B+…+base` on demand. **Mixin slot names must be unique** (prefix them): CLOS *merges* same-named slots when classes combine, so two mixins with a `subscribers` slot would cross their lists — hence the `debounce-*`/`throttle-*`/`persist-*` prefixes. `combined-class` takes `(base mixins)`, sorts the mixins by name (any permutation → one memoized class) and lazily defines the class via `eval`+`defclass`; `cell-value-source`/`cell-mixins` decompose a cell and `morph-cell`/`add-mixin`/`remove-mixin` recombine it (so `set-external`/`set-async` preserve mixins across a value-source change). Add a new axis by defining a mixin (overriding some generic — primary, `:after`, `:around`, or a fresh one) and listing it in `*mixin-classes*`.
- **Attributes** (orthogonal, *not* classes — ref-set registries on the sheet, so they layer onto any kind): **volatility** (`set-volatile`/`volatile-p`, folded into every sweep's seeds by `recompute-closure`) and **frozen** (`set-frozen`/`frozen-p`, where `compute-cell` short-circuits and holds the cell's value).

A `sheet` is a `defstruct` — a ref→cell hash table, an `environment` alist of `(symbol . value)` constants, a `volatiles` ref-set registry, and a recursive `lock`. (Note: `cell` the class and `cell` the in-formula reader function share one symbol.)

**Extension seams** (`eval.lisp` defines the generics; `taxonomy.lisp` the overrides) — `compute-value (cell sheet ref)` produces a cell's value (base method runs `eval-formula`); `cell-swept (cell sheet ref)` runs once per cell computed in a sweep, after the compute loop, **only for cells that did not error** (an errored cell has no settled value for the `:after` sinks), for post-sweep notification (base method inert); `cell-writable-p (cell &optional new-formula)` gates the public mutators, seeing the proposed formula (base method `t`); `note-set (cell sheet ref new-formula actor time)` fires on the set path just before the formula is replaced, carrying the `*actor*` (bound by `with-actor`) and a `*audit-clock*` timestamp (base method inert; mixins hook it `:after` so several compose). Add a value-source kind as a subclass with a `compute-value` method; add a cross-cutting behavior as a mixin (overriding a *different* generic, so combination classes never collide) listed in `*mixin-classes*` — rather than editing the core. `combined-class` then handles any arity of mixin over any value source.

**Formula standard library** (`stdlib.lisp`) — helpers callable from a formula, exported so they resolve unqualified when the caller's package uses `cellisp`. A formula is arbitrary Lisp, so CL's `min`/`max`/`if`/… already work on explicit args; the stdlib adds spreadsheet conveniences CL lacks: numeric aggregates that ignore non-numbers via `flatten-numbers` (`minimum`/`maximum`/`product`/`median`, the same family as `sum`/`average`/`cnt` in `eval.lisp`), predicate-filtered aggregates (`countif` — applies its predicate defensively so it never aborts on a value it errors on; `sumif`/`averageif` over numbers), 2D range access (`grid`, sharing `resolve-range` with `cells` but keeping row structure), lookups (`match`/`lookup`/`vlookup`/`hlookup` over the lists `cells`/`grid` return), `blankp`, and the `iferror` macro (a `handler-case` yielding a default; note the wrapped form's precedents are still recorded because `cell` notes the precedent *before* the read can signal, so recovery re-fires). Range reads tolerate gaps by default: `cells`/`grid` read an *empty* cell as `nil` (blank) via `read-cell-blank`, so `(sum (cells …))` works on a sparse range (the numeric aggregates ignore `nil`) and `grid` keeps its shape with `nil` holes. An *existing* cell that holds an error still propagates it through a range; a **single-cell** `(cell "X")` read of an empty cell still signals `unbound-cell` (kept strict to preserve error propagation and the recovery model). `safe-cells` is the fully-tolerant variant — it swallows errored cells too (empty *and* error → omitted), for `(sum (safe-cells …))` over partly-broken ranges. Dependencies on blank cells are recorded either way, so filling a gap re-fires the reader. Also here: `sortv`/`filterv`/`uniquev` (dynamic-array helpers over lists), text (`concat`, `left`/`right`/`mid`, `upper`/`lower`/`trim`, `substitute-text`, `text-length`, via `as-text`), and dates as universal-time integers (`date`/`year`/`month`/`day`/`weekday`/`now`).

**Evaluation & the dependency graph** (`eval.lisp`) — the core. Five dynamic vars drive recalculation: `*sheet*` (sheet being evaluated), `*eval-stack*` (refs currently on the DFS stack, for cycle detection), `*collected-precedents*` (a hash table that, when bound, records which cells the running formula touches), `*fresh*` (a hash table, bound for one recompute sweep, of refs already computed this sweep), and `*changed*` (a hash table, bound for a sweep, of refs whose value/error actually changed — for propagation short-circuiting).

- Precedents are **rediscovered on every recompute** by evaluating the formula with `*collected-precedents*` active — `cell`/`cells` call `note-precedent`. This is deliberately dynamic so conditional references (`(if ... (cell "A1") (cell "B1"))`) track only the branch actually taken.
- `evaluate-ref` pulls a cell's value on demand, recursing into precedents depth-first. It short-circuits a ref already in `*fresh*` (reusing its value, or re-signaling its stored `err`) — so within one sweep each cell computes at most once, and a diamond/DAG doesn't recompute shared precedents per reader. The `*fresh*` check sits *before* the cycle check; a cell mid-computation isn't fresh yet, so a re-entrant read still hits `*eval-stack*` and signals `cyclic-reference`.
- `compute-cell` evaluates the formula, commits `value`/`err`, then — in an `unwind-protect` cleanup that runs **even when the formula errors** — calls `update-dependency-links` to reconcile the bidirectional back-links and marks the ref in `*fresh*`. Committing links on error is essential: otherwise an errored cell drops out of the graph and never recomputes when its inputs later recover.
- `eval-formula` runs the formula: a bare symbol resolves against the environment alist (else self-evaluates); a cons is real Lisp — `eval`'d directly when the sheet has no environment, otherwise run through a `compile`d lambda with `let` bindings. That lambda is **compiled once and cached** on the cell (`compiled`/`compiled-from` via `cell-thunk`), keyed by formula identity (`eq`) — recompiled only when the formula changes, since the environment is assumed fixed per sheet (mutating `sheet-environment` after a formula has run leaves cached thunks stale). Environment values are spliced in **quoted**, so a list- or symbol-valued constant is treated as data, not evaluated as a form. **Formulas are unsandboxed Lisp** — treat any formula input as arbitrary code execution.

**Public API** (`api.lisp`) — `set-cell`/`clear-cell` mutate, then call `recompute-closure`. `affected-closure` collects a seed plus its transitive dependents via DFS over the `dependents` graph; `topological-order` sorts them precedents-first; `recompute-closure` walks that order and recomputes a cell only when it is a seed/volatile or a precedent's value **changed** — `compute-cell` records changes into `*changed*` (covering on-demand pulls too), so a cell that recomputes to an unchanged value **short-circuits its whole subtree** (a stable "gate" prunes everything below it). Volatile refs are folded into the seeds so they refresh every sweep. `*fresh*` still guarantees each cell computes at most once, and `evaluate-ref` still pulls precedents on demand, so out-of-order reads stay correct; skipped (unchanged) cells are marked fresh so readers reuse them, and `cell-swept` fires only for cells actually recomputed. Per-cell errors are stored on the cell and swallowed during a sweep so one broken cell doesn't abort the whole recompute; `set-cell` re-signals only its *own* cell's error. `set-cells` takes a list of `(designator formula)` pairs, installs *all* formulas before recomputing, then runs a single `recompute-closure` over the combined seeds — so batch cells may reference each other in any order (forward refs resolve with no transient error) and shared dependents recompute once. It does *not* re-signal (errors stay on the cells); it returns the resulting values in input order. `set-cell` also takes a sticky `:volatile` keyword — supply it to add/remove the ref from the volatile registry (omit it and volatility is left unchanged); `set-volatile` toggles it independently on any existing cell, and `volatile-p`/`volatile-refs` expose that state. Reads never raise: `get-value` returns `(values value error-or-nil)` (empty cell → `nil, nil`), so callers inspect the second value rather than handling a condition; `get-formula` returns the stored form. `recalc`/`recalc-all` force recomputation on demand, and `dependents`/`precedents`/`map-cells` expose the graph for introspection. A sheet may carry a **change hook** (`set-change-hook`): after every sweep `recompute-closure` hands it the row-major-sorted refs whose value/error actually changed (gathered from `*changed*`, only when a hook is installed — the hot path pays nothing otherwise), plus any `:extra-changed` refs the caller forces in (`clear-cell` reports the ref it just removed, which never recomputes itself). This is the UI repaint set: empty when an edit changes nothing (short-circuit), covering every edit path since they all funnel through `recompute-closure`. It's a live closure, so it isn't serialized. `used-range` (tight bounding box as a `(tl . br)` cons, or nil) and `sheet-dimensions` (rows, cols as two values) give a renderer the grid extent — both ignore pure dependency-placeholder cells (`%cell-content-p`), so a formula merely referencing an empty `Z9` doesn't balloon the range.

**Workbooks / cross-sheet references** (`workbook.lisp` + cross-sheet logic threaded through `eval.lisp`/`api.lisp`) — a `workbook` is an ordered, case-insensitive name→sheet map; `add-sheet` stamps a sheet's `workbook` back-ref and `name`. A sheet from `make-sheet` has `workbook` = nil and runs the **exact single-sheet path** — every cross-sheet branch guards on `sheet-workbook`, and `compute-cell` only allocates the foreign table for a workbook sheet, so single-sheet compute is allocation-identical. In a formula, `"Data!A1"` is split by `split-sheet-designator`; `cell`/`cells` resolve the target sheet in the current sheet's workbook and read it via `read-cell-value`, which for a *foreign* target does a **non-recursive read of the producer's cached value** (`read-foreign`) — it does *not* recurse into the other sheet's evaluation, so `*fresh*`/`*eval-stack*`/`*changed*` stay per-sheet and bare-ref-keyed (a sweep only ever touches one sheet's cells). Cross-sheet edges are tracked symmetrically to intra-sheet ones: a consumer cell's `foreign-precedents` (list of grefs — `(sheet . ref)` conses) and each producer sheet's `foreign-dependents` table (local-ref → consumer grefs), reconciled by `update-foreign-links` each compute. Propagation: `recompute-closure` = a local sweep (`recompute-local`, which returns the changed refs and fires the change hook) followed, when the sheet is in a workbook, by `cascade-foreign` — a **fixpoint cascade** that recomputes foreign consumers of the changed cells wave by wave (producers before consumers, so foreign reads see settled values), with a per-cell revisit cap (`~sheet-count`) that breaks genuine cross-sheet reference cycles by flagging the offending cells `cyclic-reference` (`%flag-cross-cycle`) rather than looping. Because one cascade can sweep a sheet several times (once per producer feeding it) and an on-demand pull in an early sweep can settle a cell's value, the per-sweep `*changed*` alone would miss propagating that change to a dependent recomputed in a *later* sweep; `recompute-closure` binds `*sticky*` (a workbook-wide set of changed `(sheet . ref)` handles) for the whole cascade, and the recompute short-circuit treats a precedent as changed if it changed in *any* sweep — so nothing falls between sweeps. `recompute-workbook` settles a whole workbook (used after load). Serialization reuses `sheet->form` per sheet (a cross-sheet formula is just its readable string); `form->workbook` rebuilds each sheet standalone (cross-refs error transiently, harmlessly), attaches them, then `recompute-workbook` settles the graph. `explain`/`explain-tree` (`explain.lisp`) walk a cell's precedent tree — sorted by ref for determinism — showing each formula and value/error and following an error branch to the deepest cell that actually failed (the root cause); cycles and repeated cells are collapsed. `set-cell`/`set-cells`/`clear-cell` log an **undo** entry (a snapshot of the affected cells' formulas, `:absent` for cells that didn't exist); `undo`/`redo` replay it with `*recording*` bound nil, and each mutation clears the redo stack. Only formula edits are undoable (structural edits and mixin/attribute changes are not). `with-transaction` groups edits: inside it the `*deferred*` dynamic var (a hash of seed refs) makes `set-cell`/`set-cells`/`clear-cell` install formulas but skip recompute and per-edit undo; on commit it pushes one combined undo entry and runs a single `recompute-closure`; on a non-local exit it rolls back to a full pre-transaction snapshot (created cells cleared, others restored). Reads inside the body see pre-commit values (recompute is deferred). Sheets also carry pure **UI metadata** the engine never reads: cell **notes** (`set-note`/`cell-note`, a ref→string table) and **merged cells** (`merge-cells`/`merged-range`/`merges`, a list of `(tl . br)` rects, overlaps refused). Both follow their cells across structural edits (`shift-registry`/`shift-merges` in `edit.lisp`) and serialize alongside names. Cell-kind drivers all live in `taxonomy.lisp` — one `set-*`/verb per axis (see the taxonomy list above): value-source (`set-external`, `set-async`/`refresh-async`/`deliver-async`), the behavior mixins (`observe`, `set-logged`, `debounce`, `throttle`, `on-threshold`, `set-stats`, `set-persist`, `set-cached`, `set-ttl`, `set-default`, `set-transform`, `set-validator`, `set-timed`, `set-retry`, `set-readonly`, `set-append-only`, `set-typed-input`, `set-versioned`, `set-audited`), and the attributes (`set-volatile`, `set-frozen`). Wrap mutations in `with-actor` to stamp the author into audit entries.

**Serialization** (`serialize.lisp`) — `write-sheet`/`read-sheet` (stream) and `save-sheet`/`load-sheet` (file) round-trip a sheet as one readable form: the `environment` and, per cell, its `formula`, its *declarative* attributes (`volatile`, `frozen`, `readonly`, `append-only`), and its *durable history* — `audited`'s log, `versioned`'s formula edits, `logged`'s value history (+ limit), and `stats`' accumulators (the non-recomputable state; transient mixin state like cache signatures/timers is skipped). Closure-config mixins round-trip **when their function is a named function (a symbol)** rather than an anonymous lambda — `funcall` accepts a symbol, so the mixins consume it transparently; `write-sheet` records `external`/`async` source, `validator`, `transform`, `sink`, and named `observers` as symbols (a live lambda is skipped, not written) plus data config (`default`, `retry`, `ttl`). Values and errors are **not** stored. `read-sheet` reconstructs in phases so semantics hold: (1) create cells and apply value-wrapping/source config, then (2) install formulas with `set-cells` (forward refs resolve, wrappers already active), (3) declarative attributes (readonly/append-only last, so locking never blocks a cell's own install), (4) durable history (add the mixin, load its slots — no recompute follows, so the restored state and now-live mixin stand), (5) reactive sinks (`persist`/observers) last, so nothing fires on load. **Out of scope by nature:** closure-bearing mixin config (external sources, validators, transforms, callbacks, schedulers) can't be written — reattach it after loading; and formulas/environment must be **readably printable**. `write-sheet` binds `*package*` to `#:cellisp` and `*print-escape*` so unqualified operators (`cell`, `+`) round-trip while unreadable objects fail at `read`.

**Structural editing** (`edit.lisp`) — `insert-row`/`delete-row`/`insert-column`/`delete-column` (1-based indices; column A is 1) move cells to shifted positions and rewrite the *static* references in every formula so they keep pointing at the same data. All four build a `ref → ref-or-:deleted` shift function and hand it to `structural-edit`, which rewrites formulas (`rewrite-formula-refs` walks the form, mapping literal ref args of `cell`/`cells` — computed refs are left as-is), re-keys the cells hash and the volatile/frozen registries, clears the now-stale adjacency lists, and rebuilds the graph and values with `recalc-all`. A reference to a *deleted* cell is rewritten to `"#REF!"`, which errors on eval (spreadsheet #REF!).

**Copy/paste** (`edit.lisp`) — `copy-cell` (one dest) and `fill-range` (a rectangle) copy a formula, shifting its **relative** references by the source→dest offset while parts marked **absolute** with `$` (`$A$1`, `$A1`, `A$1`) stay fixed. `parse-ref` strips `$` (it's a copy-time annotation, ignored when resolving a ref), and `parse-ref-parts`/`render-ref` preserve the markers through a copy. Both structural-edit and copy share `map-formula-refs` (the literal-ref walker); structural-edit shifts every ref (cells physically move, `$` preserved), copy shifts only the relative parts. **Spill** (`spill sheet anchor formula`) evaluates an array-valued formula (a list → a column, a list of lists → a 2D block) and populates a rectangle from the anchor, each cell indexing the formula (so it tracks the formula's inputs; shape is fixed at spill time). The class-changing ones promote in place via `change-class` and compose freely (e.g. an external + volatile + observed + readonly + logged cell). Blocked mutations signal `readonly-cell`; bad values signal `invalid-value`. The full exported surface is the `:export` list in `package.lisp` — including the in-formula helpers `cell`, `cells`, `sum`, `average`, `cnt`.

## Conventions

- Conditions form a hierarchy rooted at `sheet-error` (defined in `sheet.lisp`, above the sheet struct): `cyclic-reference`, `unbound-cell`, `cell-eval-error` (wraps a non-sheet error raised inside a formula, preserving the `original`). Signal these rather than raw `error` for anything a formula author could trigger. `compute-cell` catches `sheet-error` and re-signals it as-is, but wraps any other `error` in a `cell-eval-error` — so formula bugs always surface as a `sheet-error` subtype.
- The in-formula aggregates (`sum`/`cnt`/`average`, all over `flatten-numbers`) **ignore non-numeric values** spreadsheet-style: `sum` of nothing is `0`, `cnt` counts only numbers, and `average` of no numbers **signals `sheet-error`** rather than dividing by zero.
- Every public entry point takes the sheet's **recursive `lock`** (`with-sheet-lock`), so readers, writers, and out-of-band `deliver-async` calls from other threads are serialized — a sheet is thread-safe at the API boundary. The lock is recursive so a callback fired mid-sweep (an observer) may re-enter the read API. Internal functions (`recompute-closure`, `compute-cell`) are *not* locked — they run inside an already-locked caller. Observer/fetcher callbacks run while the lock is held, so they must not block on another thread that needs the sheet.
- The exported symbol list in `package.lisp` is the public surface — update it when adding user-facing functions.
